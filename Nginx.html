# 中间件架构

# 中间件技术可以带来什么

  • 间件是在操作系统功能范围外为应用提供服务的多用途软件。任何位于内核和用户应用之间的软件都可以是中间件。中间件不提供传统应用的功能，而是将软件与其他软件衔接。由于中间件能够让数据从一个应用流动到另一个中，因此把它比作输水管最为贴切。

# Nginx简述

  • Nginx是一个高性能的且开源、可靠的HTTP中间件、代理服务

# 常见的HTTP服务

  • HTTP
    • -Apache 基金会
  • IIS
    • 微软
  • GWS
    • -Google

# Nginx的优点

# 什么是IO复用(IO多路复用epoll)

  • 请求、IO流 -> socket -> 多线程

  • 具体

    • 多个描述符的I/O操作都能够在一个线程中内并发交替的顺序完成，这就叫I/O多路复用，这里的复用是指的复用同一个线程

# 轻量级

  • 通常说的轻量级的标准
    • 功能模式少
    • 代码模块少

# CPU亲和(affinity)

  • 老牌服务器

    • Apache
      • 老牌的服务器软件
      • 功能多，并且配置丰富、稳定
    • Nginx
      • 解决并发问题
      • 简单小巧
      • 并发量高

  • Nginx主要用途

    • Web服务器
    • 代理服务器
    • 邮箱服务器
    • immp

  • 数据库

    • mysql
    • SQlserver
    • oracle
    • DB2
    • mariaDB

  • NOSQL

    • memcached
    • redis
    • mongdb

# 负载均衡

  • 负载均衡服务器，分发请求到不同的服务器，使流量平均分配

  • 服务器死机都成为宕**(dang)**机

  • 负载级别

    • 软件级别
      • 性能好
      • 价格高
    • 软件级别
      • 性价比高
    • lvs
      • linux下分发软件
    • nginx
      • upstream功能分发

# 资源服务器

  • 资源服务器储存静态资源、css、js、图片、视频
  • 一般此服务器会有硬盘（SSD），读写速度快
  • 带宽高

# 代理服务器架构

# 为什么需要反向代理服务器

  • 由于浏览器安全策略限制，跨域请求都不能够访问
  • a ->× b
  • 代理服务器
    • a -> c -> b

# 正向代理和反向代理

  • 服务器 -> 外网服务器 -> 个人

  • 代理其实就是叫别人去帮你做你不能做的事情

  • 代理服务器具有代理到对应服务器的功能，有具有缓存的功能，、

# 代理服务器的种类

  • nginx
  • squid
  • varnish

# Nginx的优点

  • 热部署

    • nginx在修改配置文件之后，不需要重启，重载操作

  • 可并发可连接

    • 相同配置的服务器，nginx和apache能连接的更多

  • 低的内存消耗

    • 相同的服务器，nginx比apache低的消耗

  • 处理响应请求很快

    • nginx处理静态方面比其他的一些服务器速度更快

# Nginx和apache

  • apache在安全和稳定性比nginx好些
  • lnmpa架构
  • ianmN架构
  • 其实都是借助各自的优点

# Nginx的安装和使用

# 环境准备

  • yum可用
  • 确认关闭iptables
  • 确认停用selinux

# 运行环境

  • 安装gcc以及其他的一些软件
yum -y install gcc gcc-c++ autoconf pcre pcre-devel make automake

# 版本

  • Nginx分为开发版（mainline）和正式版（stable）以及以往的历史版本（legacy）
  • 版本中的pgp是代码效验的意思，其为验证安装是否为原版的链接，防止使用被二次修改的Nginx安装包

# 安装

  • 你可以使用离线安装，也可以选择安装使用 yum安装

# 在线安装

  • 点击官网下面的 Pre-Built Packages下的链接
  • 选择对应的系统版本，如RHEL/CentOS
  • 复制对应的yum源，如
[nginx-stable]
name=nginx stable repo
baseurl=http://nginx.org/packages/centos/$releasever/$basearch/
gpgcheck=1
enabled=1
gpgkey=https://nginx.org/keys/nginx_signing.key
  • 编辑保存
  • 查找对应的Nginx安装包
# 查找对应nginx版本
yum list | grep nginx
# 直接安装nginx 版本
yum install nginx
# yum 测试安装
nginx -v
# yum 测试是否成功
nginx -V

# 基本安装目录

  • 安装目录讲解

rpm -ql nginx

# 安装编译参数

  • 命令
# 得到nginx安装的详细配置文件地址
nginx -V 
# 设定Nginx进程启用的用户和组用户 
--user=nginx
--group=nginx
# 设置额外的参数将被添加到cflags中
--with-cc-opt=parameters
# 安装编译参数，设置附加的参数，链接系统库
--width-ld-opt=parameters

# Nginx.conf

  • 注意编译nginx中出现中断会产生一个临时空文件，nginx.conf.swp文件，导致再次编译nginx.conf会编译nginx.conf.swp
    • 可以删除该文件
    • 也可以继续编译该文件，并保存为最终的nginx.conf文件
# 在nginx中的conf.d文件夹中的conf配置也会被读取到配置中
include /etc/nginx/conf.d/*.conf;
  • 在nginx.d文件夹中有一个default.conf文件，是一个默认conf文件

# Nginx.conf配置

// 一般我们都会把限制语句放在前面，防止后面的配置执行，拦截超出请求

limit_conn_zone $binary_remote_addr zone=conn:1m;
