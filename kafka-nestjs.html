kafka-nestjs

# kafka基础概念

# Topic

kafka消息存放在Topic(主题,类似于数据库中的表)中。

# Partition

Topic包含多个Partition(分区),Partition可以存放到不同的服务器上。

# Record

每条Record(消息)存放到Partition中，每条Record有一个Offset(偏移量)以记录Record的位置，kafka可以通过Offset提取Record(不能通过Record的内容进行检索和查询)，Offset在每个分区中是唯一且递增的，不同分区的Offset可以重复。kafka中的Record以键值对(key-value)存储，key值为空则以轮询的方式写入不同分区中，相同key的Record将被写入到同一个分区。

# Replication

kafka可以设置Replication(副本)数量(包括主分区在内的副本数量)提升数据可靠性,Replication存在一个Leader(主分区),数据的读写由Leader完成，其余Follower(除Leader外的副本)只负责从Leader中复制数据，保持数据一致。ISR是同步副本集合(可以保持正常同步的副本集合)，若某个副本不能正常同步数据或者落后太多，kafka会将该副本剔除出ISR，直到该副本恢复正常再添加回ISR。

# Broker

Broker(消息代理)，负责处理Record的读写，通常会在每个服务器上启动一个Broker实例(一台服务器就是一个Broker)

# 生产和消费

  • kafka用户身份分为生产者和消费者。
  • 生产者向kafka写入数据，消费者消费(可以理解成拉取)kafka数据。
  • 一个消费者(消费者组)可以消费多个分区(不特殊指定消费所有分区)，一个分区可以被多个消费者(消费者组)消费，但是一个分区不能同时被同一个消费者组里的多个消费者消费
  • 同一个生产者发送到同一个分区的消息，可以保证顺序(先发送的Offset比后发送的Offset小)；同一个生产者发送到不同分区的消息不能保证顺序
  • 消费者按照消息按照同一个分区里的Offset顺序(不一定连续，但保证顺序)消费。若一定要保证所有消息的顺序和连续性，可以只使用一个分区
  • 提高生产者吞吐量的方法
    • 调整间隔时间(kafkajs不支持)
    • 调整批次大小(kafkajs不支持)
    • 压缩数据
    • 设置缓冲区大小(kafkajs不支持)

# 部署(3个节点)

注意版本，太高了生态跟不上(bitnami/kafka:3.6版本kafkajs跟不上)

docker pull bitnami/zookeeper:3.7.0
docker pull bitnami/kafka:2.6.0
mkdir -p /docker/kafka/data/kafka-{0,1,2}
chmod -R 777 /docker/kafka/data 
docker network create kafka-network --driver bridge
vim /docker/kafka/docker-compose.yml

version: '3'
networks:
  kafka-network:
    driver: bridge
services:
  kafka-zookeeper:
    image: 'bitnami/zookeeper:3.7.0'
    container_name: kafka-zookeeper
    hostname: kafka-zookeeper
    restart: always
    volumes:
      - /docker/kafka/zookeeper:/bitnami/zookeeper 
    networks:
      - kafka-network
    environment:
      - ZOO_ENABLE_AUTH=yes
      - ZOO_SERVER_USERS=zoo
      - ZOO_SERVER_PASSWORDS=zoo
      - ZOO_CLIENT_USER=zoo
      - ZOO_CLIENT_PASSWORD=zoo
    ports:
      - '2190:2181'
  kafka-0:
    depends_on:
      - kafka-zookeeper
    image: 'bitnami/kafka:2.6.0'
    container_name: kafka-0
    hostname: kafka-0
    restart: always
    volumes:
      - /docker/kafka/data/kafka-0:/bitnami/kafka
      - /var/run/docker.sock:/var/run/docker.sock
    networks:
      - kafka-network
    ports:
      - '9093:9093'
    environment:
      - KAFKA_BROKER_ID=1
      - KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP=INTERNAL:PLAINTEXT,CLIENT:PLAINTEXT
      - KAFKA_CFG_LISTENERS=INTERNAL://kafka-0:9092,CLIENT://kafka-0:9093

# 部分environment解释

  • KAFKA_ADVERTISED_HOST_NAME: 这是 Kafka 服务器的主机名或 IP地址，用于通知生产者和消费者连接到该地址。
  • KAFKA_ENABLE_KRAFT: 这个参数启用 Kafka Raft（KRaft）模式，它是 Kafka 2.8.0版本引入的一种新的高可用性模式。
  • KAFKA_KRAFT_CLUSTER_ID: 这是 KRaft 集群的唯一标识符，用于将 Kafka 服务器加入特定的 KRaft集群。
  • KAFKA_CFG_PROCESS_ROLES: 这个参数指定 Kafka 服务器的角色，其中 "broker" 表示服务器是一个普通的Kafka 节点，"controller" 表示服务器是集群的控制器。
  • KAFKA_BROKER_ID: 这是 Kafka 服务器的唯一标识符，用于在集群中区分不同的服务器。
  • KAFKA_CFG_NUM_PARTITIONS: 这个参数指定每个主题的默认分区数。分区是 Kafka中的消息存储单元，用于实现消息的并行处理。
  • KAFKA_CFG_OFFSETS_TOPIC_REPLICATION_FACTOR:这个参数指定偏移量主题的副本因子，偏移量主题用于存储消费者的偏移量信息。
  • KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP:这个参数定义了不同监听器的安全协议映射关系。在这个例子中，"CLIENT"、"EXTERNAL" 和 "CONTROLLER"分别映射到 "PLAINTEXT" 协议，表示这些监听器使用明文传输。
  • KAFKA_CFG_LISTENERS: 这个参数定义了 Kafka监听器的配置。在这个例子中，有三个监听器分别使用不同的协议和端口号。
  • KAFKA_CFG_ADVERTISED_LISTENERS: 这个参数指定了广告监听器的配置，用于告知生产者和消费者连接到 Kafka的地址和端口号。
  • KAFKA_CFG_INTER_BROKER_LISTENER_NAME: 这个参数指定了经纪人之间通信使用的监听器名称，其中"CLIENT" 表示使用客户端监听器。
  • KAFKA_CFG_CONTROLLER_LISTENER_NAMES:这个参数指定控制器监听器的名称，用于控制器与其他经纪人之间的通信。
  • KAFKA_CFG_CONTROLLER_QUORUM_VOTERS: 这个参数指定了控制器选举过程中的投票者信息，包括其 ID和监听器的地址。
  • ALLOW_PLAINTEXT_LISTENER: 这个参数允许使用明文传输的监听器进行连接。
  • KAFKA_AUTO_CREATE_TOPICS_ENABLE:这个参数启用自动创建主题的功能，当生产者发送消息到不存在的主题时，将自动创建该主题。
  • KAFKA_CFG_INITIAL_BROKER_REGISTRATION_TIMEOUT_MS:这个参数指定初始经纪人注册的超时时间，以毫秒为单位。在集群启动期间，经纪人必须在指定的时间内完成注册。

import { Injectable, OnApplicationShutdown, OnModuleInit } from '@nestjs/common';
import { EachMessageHandler, Kafka, Producer, logLevel } from 'kafkajs'
import * as snowId from 'simple-flakeid';
const workers = 1;
const gen1 = new snowId.SnowflakeIdv1({ workerId: workers });

@Injectable()
export class K implements OnModuleInit, OnApplicationShutdown {
    private readonly kafka = new Kafka({
        // 为了避免混淆和冲突，最好还是让每个实例或进程的clientId在全局范围内都是唯一的
        clientId: gen1.NextBigId().toString(),
        brokers: [
            '192.168.2.200:9093',
            '192.168.2.200:9193',
            '192.168.2.200:9293',
        ],
        connectionTimeout: 3000, // time in ms default value is 1000
        requestTimeout: 25000, // default 30000 ms
        retry: {
            initialRetryTime: 100,
            retries: 8
        },
        logLevel: logLevel.INFO,
    })

    /**
     * maxInFlightRequests: 1和idempotent: true保证 EOS
     * 简单来说这样配置使得Kafka和Kafka应用能够保证每条消息被精确地传递一次，而不会被重复或丢失1。这确保了计算结果的准确性
    */
    private readonly producer = this.kafka.producer({
        idempotent: true,
        maxInFlightRequests: 1,
        /**
         *全局唯一，每个生产者的transactionalId必须是唯一的 
         建议使用雪花id
        */
        transactionalId: gen1.NextBigId().toString(), 
    })

    public send: Producer['send'] | null = null
    public transaction: Producer['transaction'] | null = null

    private readonly consumer = this.kafka.consumer({
        groupId: 'consumer-id', // 消费者组id，相同的id不会重复消费
        minBytes: 10, // 每批次最小抓取大小 默认1字节
        maxBytes: 5 * 100, // 每批次最大抓取大小 默认50m字节
        maxWaitTimeInMs: 100, // 一批数据最小值未达到的超时时间 默认500ms
    })

    private readonly consumerFnMap = new Map<string, EachMessageHandler>()

    async subscribe(topic: string, fn: EachMessageHandler) {
        // if(this.consumers.has(topic)) {
        //     return
        // }
        await this.consumer.subscribe({
            topic,
            fromBeginning: true
        })
        this.consumerFnMap.set(topic, fn)
    }
    
    async onModuleInit() {
        await Promise.all([
            this.producer.connect(),
            this.consumer.connect()
        ])

        this.send = this.producer.send
        this.transaction = this.producer.transaction

        this.consumer.run({
            autoCommit: true, // 自动提交offest 默认true
            autoCommitInterval: 1000, // 自动提交offest间隔 默认5000ms
            eachMessage: async (parm) => {
                this.consumerFnMap.get(parm.topic)?.(parm)
            }
        })
    }

    async onApplicationShutdown(signal?: string) {
        await Promise.all([
            this.producer.disconnect(),
            this.consumer.disconnect()
        ])
    }
}

import { Injectable, OnModuleInit } from '@nestjs/common';
import { K } from './k.service';
import { ProducerRecord } from 'kafkajs';

@Injectable()
export class KafkaService implements OnModuleInit {
    private testTopic = 'test-topic'
    private testTopic2 = 'test-topic-2'
    constructor(
        private readonly k: K
    ) {}
    onModuleInit() {
        this.k.subscribe(this.testTopic, async (value) => {
            console.log('已消费测试主题1消息')
            console.log(`tipic:${
                value.topic
            }; message:${
                value.message.value.toString()
            }; partition:${value.partition}`)
        })
        this.k.subscribe(this.testTopic2, async (value) => {
            console.log('已消费测试主题2消息')
            console.log(`tipic:${
                value.topic
            }; message:${
                value.message.value.toString()
            }; partition:${value.partition}`)
        })
    }
    async send(record: ProducerRecord) {
        // 以事务的方式提交
        const transaction = await this.k.transaction()
        try {
            await transaction.send(record)
            await transaction.commit()
            return true
        } catch (error) {
            console.log("transaction error:")
            console.log(error)
            await transaction.abort() // 回滚
            return false
        }
    }
    hello() {
        return 'kafka hello'
    }
}

import { Controller, Get, Query, Render } from '@nestjs/common';
import { KafkaService } from './kafka.service';
import { CompressionTypes } from 'kafkajs'

import * as snowId from 'simple-flakeid';
const workers = 1;
const gen1 = new snowId.SnowflakeIdv1({ workerId: workers });

@Controller('kafka')
export class KafkaController {
  constructor(
    private readonly kafkaService: KafkaService,
  ) {
    
  }

  @Get('html')
  @Render('test.ejs')
  testHtml() {
    return {
      name: 'test html'
    }
  }

  @Get('sendToKafka')
  async sendToKafka(@Query('topic') topic: string, @Query('msg') msg: string) {
    const value = {
      topic,
      // 随机重复，放到不同分区(有3个分区可分配)
      messages: new Array(1+Math.floor(Math.random() * 3)).fill(1).map((_, idx) => ({
        value: `${msg} --- repeat ${idx+1}`,
        /**
         * 分区选填
         * 若未设置
         *   如果设置了key，则按照key的hash值对分区数进行取模
         *   如果未设置key，则粘性分区，即：随机选择一个分区，并一直使用该分区直到批次数据满了或者时间到了再随机选择一个分区(和上一次不同)重复刚才动作
        */
        partition: idx % 3, 
        key: gen1.NextBigId().toString() // 选填，有partition基本不写
      })),
      // compression: CompressionTypes.GZIP // 选择压缩方式(还有其他方式)。压缩可以减小传输量，但会增加cpu消耗，特别是js不擅长cpu密集任务，是否设置需要权衡
    }
    const ok = await this.kafkaService.send(value)
    return {ok}
  }
  
  @Get()
  hello() {
    return this.kafkaService.hello()
  }
}
